# シフト表印刷ツール 要件定義書

最終更新：2026-02-16

---

# 1. 目的

月次シフト表を A4 用紙に印刷できるツールを作成する。

## 前提条件

- クライアントPCには Node.js / 開発環境は存在しない
- 静的ファイルのみで動作すること
- クライアントがコードを直接編集せずに運用できること
- 将来的な人員増減・ローテ変更に対応可能であること

---

# 2. 配布・動作環境

## 2.1 配布形式

配布物は以下の3ファイルのみとする。

- index.html
- bundle.js
- data.js

## 2.2 実行方法

- クライアントは index.html をブラウザで開く
- ブラウザの印刷機能で出力する

## 2.3 開発環境

- React
- TypeScript
- Tailwind CSS
- esbuild
- CSSは bundle.js に埋め込み
- 配布先ではビルド不要

# 3. シフト生成ロジック仕様

## 3.1 基本方式

### 絶対基準方式を採用

任意年月のシフトは以下の手順で決定する。

1. 表示対象月の1日を取得する
2. その日に適用されるローテーション定義（roster）を選択する
3. roster の適用開始日（from）からの差分日数（UTC基準）を算出する
4. ローテーションを日数分進めて担当を決定する

担当者の位置は次の考え方で算出する。

開始担当者のインデックスに経過日数を加算し、名簿人数で割った余りをその日の担当者インデックスとする。

## 3.2 うるう年対応

- 日数差分は UTC基準で算出する
- 月日数およびうるう年は自動対応する

## 3.3 パフォーマンス

- 計算対象は表示月のみ（最大31日）
- 年が経過しても処理時間は一定
- パフォーマンス劣化は発生しない

---

# 4. ローテーション管理仕様

## 4.1 rosters 配列方式

複数のローテーションを期間単位で管理する。

各ローテーション定義は以下の情報を持つ。

- 適用開始日（from）：YYYY-MM-01（月初固定）
- 名簿（names）：ローテーション順
- 開始担当（starts）：from当日の各行の担当者

## 4.2 切替ルール

- ローテ変更は月初のみ適用する
- 月途中の変更は運用で吸収し、翌月1日から反映する
- 過去のデータは変更しない（履歴保持）

## 4.3 roster 選択ロジック

対象月の1日に対し、以下を満たすローテーションを採用する。

- from が対象日以前である
- 条件を満たす中で最も新しい from を選択する

## 4.4 roster 編集方針

- rosters に新しい定義を追加して管理する
- from は UI上で年月選択のみ（手入力不可）
- from の重複は禁止する
- rosters は自動で昇順ソートする

---

# 5. 表示仕様

## 5.1 印刷仕様

- 用紙：A4
- 向き：横（landscape）
- 余白：10mm
- 1ページ完結

## 5.2 レイアウト

2段構成とする。

- 上段：1〜16日
- 下段：17日〜月末
- 段間：余白のみ（区切り線なし）

## 5.3 行構成

- 日付
- 曜日
- フロアA
- フロアB
- 外回り
- 休み1
- 休み2
- 休み3

---

# 6. 装飾ルール

## 6.1 曜日

- 土曜：青文字
- 日曜：青文字

## 6.2 自分（me）の強調

- フロアA / フロアB / 外回りの場合
  背景黄色 + 太字

- 休みの場合
  背景黄緑 + 太字 + 赤文字

## 6.3 外回り → 3日休み

- ローテーション自体は変更しない
- 外回り担当の翌日から3日間の休みセルを強調表示する（highlight）
- 将来的に override モードを追加可能

---

# 7. 年月選択UI仕様

## 7.1 入力方式

- 年：ドロップダウン（現在年 -1 〜 現在年 +3）
- 月：1〜12
- 前月ボタン
- 次月ボタン
- 印刷ボタン

## 7.2 初期値

- 現在の年
- 現在の月

---

# 8. 簡易エディタ仕様

## 8.1 目的

クライアントがコードを直接編集せずに data.js を生成できるようにする。

## 8.2 編集可能項目

- 自分の名前（me）
- rosters 一覧
- 名簿（1行1名入力）
- 開始担当（名簿から選択式）

## 8.3 data.js生成

- ボタン押下で data.js 文字列を生成する
- コピーボタンでクリップボードへコピーする
- 以下の注意文を表示する
  既存のdata.jsを丸ごと置き換えてください

---

# 9. バリデーション仕様

## 9.1 名簿

- 空不可
- 重複不可
- 空行除去

## 9.2 開始担当

- 名簿内からのみ選択可能
- 名簿変更時に整合性チェックを行う

## 9.3 rosters

- from 重複禁止
- from は月初固定
- roster最低1件必須

---

# 10. 運用フロー

1. エディタでデータ編集する
2. 「data.js生成」を押す
3. 出力テキストをコピーする
4. 配布フォルダ内の data.js を丸ごと置き換える
5. index.html を再読み込みする

---

# 12. 非機能要件

## 12.1 動作環境

- ブラウザ単体で動作すること
- ネットワーク接続不要
- Node.js等の実行環境は不要

## 12.2 パフォーマンス

- 表示対象月（最大31日）のみ計算する
- 年数が経過しても処理時間は一定
- 画面操作に対し即時応答すること

## 12.3 安定性

- 入力ミスが発生しても画面がクラッシュしないこと
- バリデーションにより不正データを防止すること
- rosterが最低1件存在することを保証すること

## 12.4 印刷品質

- A4横1ページで収まること
- レイアウト崩れが発生しないこと
- 余白・段構成が常に維持されること
